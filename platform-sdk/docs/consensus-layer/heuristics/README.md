# Heuristics — Index

Conditional diagnostic and review knowledge of the form
**observable symptom → suspected cause → validation**.

This is the catalog of experienced-engineer pattern-match knowledge that the team
otherwise carries as tribal lore. Entries are independent and self-contained: one
suspected cause and one validation path per entry. Many entries may share an observable
symptom — they are linked by the `symptoms` tag, not nested.

- Entry format: see `FORMAT.md`.
- Allowed symptom values: see top-level `symptoms.md` (`SYM-NNN`).

Entries are born `unvalidated` and stay that way until a real instance confirms them.
Treat `status` as load-bearing.
